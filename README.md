# GitHub flow
* **master**  
直接mergeしないこと。
* **develop**  
PRは基本ここにだす。  
ここで動くことを確かめたらmasterにmergeしていく。  
* **feature**  
まずはissuesに何を変更するのかを書く。  
developや他のfeatureからブランチを切る。    
命名は"feature/issueXX/hogehoge"というようにissue番号と要約を載せる。
